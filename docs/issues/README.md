# Documentation Issues

Issues discovered during documentation research. Each file in this directory tracks issues found by documentation agents for a specific topic area.

## Format

Each issue file should use this format:

```markdown
# Issues: [Topic Area]

## [Issue Title]

- **Severity**: critical | major | minor | cosmetic
- **Area**: code | docs | config | schema | API
- **Description**: What's wrong
- **Location**: File path or area of codebase
- **Suggestion**: How to fix (if known)
```

## Status

These issues are for future addressing and are not blocking documentation.
